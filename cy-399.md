## 项目主要需求

### 项目概述
**项目类型：命令行工具**
搭建一个摄影约拍平台 CLI 发布工具，摄影师可通过命令行快速发布约拍档期、查看预约请求、管理拍摄订单，适合偏好终端操作的摄影从业者。

### 功能需求

1. **摄影师注册/登录**：通过命令行输入用户名、密码完成注册和登录，Token 本地持久化存储在 `~/.photo-cli/config.json`。
2. **发布约拍档期**：命令行输入可约日期、时段（上午/下午/全天）、拍摄类型（写真/婚纱/证件照/商业/活动跟拍）、价格、服务城市，批量发布多个日期的档期。
3. **查看预约请求**：列出所有待确认的预约请求，显示客户姓名、预约日期、拍摄类型、备注，支持确认/拒绝操作。
4. **订单管理**：查看所有订单列表，按状态（待确认/已确认/拍摄中/已完成/已取消）筛选，标记订单完成。
5. **收入统计**：查看月度/年度收入汇总，按拍摄类型分类统计，命令行表格形式展示。
6. **个人资料管理**：更新摄影师简介、作品集链接、服务报价、擅长风格标签。
7. **交互式菜单**：主菜单使用 inquirer 风格的交互式选择，支持上下箭头导航和快捷键操作。

### 技术要求

- 语言：Node.js + TypeScript
- CLI 框架：commander.js（命令定义）+ inquirer（交互式提示）+ chalk（终端颜色输出）+ cli-table3（表格展示）
- HTTP 客户端：axios（调用后端 API）
- 配置存储：`~/.photo-cli/config.json`（Token、用户信息）
- API 通信：所有数据通过 RESTful API 从后端获取（后端 API 地址可配置，默认 `http://localhost:3000`）
- 代码规范：严格遵循单一职责原则，每个文件只负责一个功能；常量（错误码、拍摄类型枚举、订单状态）统一维护在 `src/constants/` 目录下；API 调用统一封装在 `src/api/`；所有工具函数封装在 `src/utils/`。

## 代码存放规则

命令行工具项目，无前端代码。

CLI 代码放在 `cli/`：

```
cli/
├── src/
│   ├── commands/           # 命令定义（按功能拆分）
```

## 部署要求

CLI 工具无需 Docker，提供安装和使用说明即可。

安装方式：
```bash
cd cli
npm install
npm run build
npm link   # 全局安装 photo-cli 命令
```

使用方式：
```bash
photo-cli login          # 登录
photo-cli register       # 注册
photo-cli schedule       # 管理约拍档期
photo-cli booking        # 查看预约请求
photo-cli order          # 订单管理
photo-cli stats          # 收入统计
photo-cli profile        # 个人资料管理
photo-cli                # 进入交互式主菜单
```
